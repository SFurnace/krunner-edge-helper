# 搜索算法说明

## 核心特性

### 1. 多关键词匹配
- 使用空格分隔多个关键词
- 所有关键词都必须匹配（在标题或文件夹中）
- 任一关键词不匹配，该书签不会出现在结果中

### 2. 匹配规则

#### 中文文本（拼音匹配优先）
- **拼音完全匹配**：95-92分
  - 例如："liushui" → "流水"
- **拼音前缀匹配**：88-82分
  - 例如："liu" → "流水"
- **拼音子串匹配**：75-69分
  - 例如："lsx" → "流水线"（在混合文本"EdgeOne 部署流水线"中）
- **原文前缀匹配**：85分
  - 例如："流" → "流水线"

#### 英文文本
- **完全匹配**：100分
  - 例如："edge" = "edge"
- **完整单词匹配（开头）**：98分
  - 例如："edge" → "Edge Browser"
- **完整单词匹配（中间）**：95分
  - 例如："edge" → "Microsoft Edge"
- **前缀匹配**：85分
  - 例如："edg" → "edge"
- **单词前缀匹配**：80-74分
  - 例如："brow" → "Browser Settings"
- **单词内子串匹配**：65-57分
  - 例如："eo" → "Edgeone"（需在单词前4个字符内）

### 3. 权重策略
- **标题权重**：1.0
- **文件夹权重**：1.0
- **URL**：不参与匹配
- 采用**最高分**策略（非平均分）
- 标题和文件夹都匹配时额外 +5 分
- 所有关键词都在标题匹配额外 +3 分
- 第一个关键词在标题开头额外 +2 分

### 4. 排序规则
1. 按分数降序排列
2. 分数相同时，按名称长度排序（短的在前）
3. 名称长度相同时，按字母顺序
